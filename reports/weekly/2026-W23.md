# 本周雷达 — 2026-W23

## 一句话总览
代理落地期，安全成本成瓶颈。

## 三个值得深挖的方向

### 1. **给编码代理做“可控运行层”：沙箱、权限、成本、审计**

**数据支撑**

- [withastro/flue]：Astro 团队出的 sandbox agent framework，说明“Claude Code 式代理框架”正在标准化。
- [microsoft/mxc]：微软做 policy-driven isolation / containment，直指“不可信代码/工具执行”的底层问题。
- [Boxes.dev] / [AppWizzy]：Product Hunt 上连续出现“在私有云/VM 里跑 Claude Code、Codex”的产品。
- [Lowfat]：HN Show HN，CLI filter 声称节省 91.8% LLM tokens；同时 [Tokenomics] 研究也在量化 agentic SWE 的 token 使用。

**为什么这是机会**

现在大家不是缺“又一个 coding agent”，而是缺一层能让团队放心用的基础设施：

- 代理会执行 shell、改代码、访问 repo、调用外部工具，安全边界模糊。
- token 成本和上下文污染开始成为真实痛点，不再只是“模型费用小优化”。
- HN 上 [Did Claude increase bugs in rsync?] 512 评论、[Ask HN: Why is the HN crowd so anti-AI?] 671 评论，说明开发者对代码质量和可控性仍然高度不信任。

缝隙在于：Cursor / Claude Code / Codex 偏个人生产力，企业需要的是**策略、隔离、日志、回滚、成本上限、合规审计**。

**与他的 SaaS 背景的契合点**

契合点很强：这是典型 B2B SaaS 控制面问题，不一定要训练模型。可以从现有工具链切入：

- GitHub / GitLab / CI / Slack / Linear / Jira 集成；
- agent session 管理、权限策略、审计记录；
- 团队级 usage / cost dashboard。

需要补的新能力：开发者基础设施、容器/沙箱、安全模型、DevSecOps credibility。

**立即可做的小动作**

1. 访谈 5 个 20-200 人工程团队的 CTO / Eng Manager：问他们是否允许 Claude Code/Codex 直接操作生产 repo，卡在哪里。
2. 做一个极小原型：代理执行前后自动生成 diff、命令日志、token 花费、风险评分，并能一键回滚 PR。

### 2. **垂直运营代理：先从电商/客服这种 ROI 清晰场景切**

**数据支撑**

- [SellerClaw]：定位“a team of AI agents that runs your stores across channels”，直接打多渠道电商运营。
- [Manus Shopify Connector]：用聊天方式 build/manage Shopify stores。
- [Cignara]：Fortune 500 grade customer support agents，客服仍是企业代理最热落地点之一。
- [Agent Mode on Arena]：泛化“real-world tasks”的代理产品出现，但边界仍宽。

**为什么这是机会**

横向通用代理太容易变成 demo；电商/客服的机会在于任务闭环清楚：

- 商品上架、标题优化、库存同步、评论回复、客服 ticket、退换货、促销活动，都有明确输入输出。
- 商家已经在 Shopify、Amazon、TikTok Shop、Klaviyo、Zendesk、Gorgias 之间来回切换，天然痛。
- 小团队愿意为“少雇 1 个运营/客服”付费，ROI 比“提升效率 10%”更好讲。

现成方案的问题：很多是 chat wrapper，能给建议但不敢真正执行；或者只能管单一平台，不能跨渠道闭环。

**与他的 SaaS 背景的契合点**

如果他过去做过 B2B SaaS / workflow / integration，这是非常贴近的方向：

- 价值不在模型，而在业务对象建模、权限、审批流、连接器、失败恢复。
- 可以从单一高频动作起步，比如“自动修复低转化商品页”或“自动处理 30% 重复客服单”。

需要补的新能力：垂直行业 know-how，尤其是电商指标、平台规则、客服质检。

**立即可做的小动作**

1. 找 10 个年 GMV 50 万-500 万美元的 Shopify 多渠道卖家，问他们每周最机械、最讨厌、最容易出错的运营动作。
2. 做一个窄原型：读取 Shopify 商品 + 客服历史，只自动生成“可审批的修改建议”，先不自动执行，测试信任门槛。

### 3. **本地优先的“工作记忆层”：让代理真正理解用户文件、应用和历史**

**数据支撑**

- [MemPalace/mempalace]：开源 AI memory system，GitHub Trending，forks 7117，说明 memory 仍是基础缺口。
- [Minimi]：Product Hunt 上的 “ambient memory for Claude”，面向现有 AI 助手补记忆。
- [Omni]：HN Show HN，本地优先的 macOS 多模态文件搜索，覆盖 text / PDF / image / audio / video。
- [Perplexity Personal Computer for Windows] / [LocalClicky]：桌面级代理开始进入本地文件、应用、语音控制。

**为什么这是机会**

代理要从“回答问题”变成“替你做事”，必须知道：

- 你的文件在哪里；
- 最近沟通过什么；
- 哪些项目、客户、任务是同一件事；
- 哪些信息不能上传云端。

现在的缝隙：ChatGPT/Claude 对个人和公司的长期上下文仍然断裂；企业知识库又太重、更新慢、权限复杂。本地优先 memory + permissioned connectors 正好卡在中间。

这不是单纯搜索产品，而是**给代理用的个人/团队上下文 API**。

**与他的 SaaS 背景的契合点**

SaaS 创始人可以借力：

- workspace onboarding；
- 权限和团队管理；
- 连接 Google Drive、Slack、Notion、GitHub、HubSpot、邮件；
- 按角色做 context pack，比如销售、客服、产品经理。

需要补的新能力：桌面端、本地索引、隐私安全、跨应用权限模型。

**立即可做的小动作**

1. 访谈 5 个重知识工作的角色：销售 AE、客服主管、顾问、产品经理、创始人助理；问他们“每天重复找哪些上下文”。
2. 做一个单人版 demo：本地索引文件 + Slack/邮件，只支持一个高价值动作，比如“生成客户会前 briefing”。

## 反直觉的 1 个观察

本周最强信号不是“大家更兴奋了”，而是“大家更不信任了”。

一边是 [Ask HN: What was your "oh shit" moment with GenAI?] 977 评论、[Anthropic recursive self-improvement] 688 评论，说明 AI 震撼感还在；另一边 [Ask HN: Why is the HN crowd so anti-AI?]、[Did Claude increase bugs in rsync?]、[Programmers will document for Claude, but not for each other] 都在讨论质量、流程、信任问题。

所以机会不在再做一个炫技 agent，而在**让 agent 可控、可验收、可追责**。这类产品看起来不性感，但更像企业会付钱的 SaaS。
